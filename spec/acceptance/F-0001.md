# F-0001: Project Setup — Acceptance Criteria

## Context
The project infrastructure must be in place before any game code can be built or deployed.

## Acceptance Criteria

### AC-1: Repository and toolchain
- [ ] Git repository initialised with `.gitignore` covering `node_modules`, `dist`, `.env`
- [ ] `package.json` present with `dev`, `build`, `preview`, `test`, `test:watch` scripts
- [ ] `npm run dev` starts a local dev server without errors
- [ ] `npm run build` produces a `dist/` artefact without TypeScript or Vite errors

### AC-2: TypeScript
- [ ] `tsconfig.json` present; `npx tsc --noEmit` exits 0 on a clean checkout

### AC-3: Testing infrastructure
- [ ] Vitest configured; `npm test` runs and exits 0 (even with no test files yet)

### AC-4: Agentic framework
- [ ] `.agentic/` directory present with `STACK.md`, `FEATURES.md`, `STATUS.md`, `CONTEXT_PACK.md`
- [ ] Git hooks installed via `git config core.hooksPath .agentic/hooks`
- [ ] `spec/FEATURES.md` lists all planned features with stable IDs

### AC-5: Deployment pipeline
- [ ] `.github/workflows/deploy.yml` present and valid YAML
- [ ] Pushing to `main` triggers a GitHub Pages deployment at `https://<org>.github.io/virtual-tree/`
