# F-0003: Isometric Terrain Rendering — Acceptance Criteria

## Context
The game world must look like a genuine isometric grid — continuous, gap-free tiles viewed from a 45° angle.

## Acceptance Criteria

### AC-1: Correct isometric projection
- [ ] Grid tiles use a 2:1 width-to-height diamond ratio (default `TILE_W=128`, `TILE_H=64`)
- [ ] `IsometricUtils.gridToWorld` produces pixel coordinates where adjacent tiles share exact edges
- [ ] `IsometricUtils.worldToGrid` round-trips correctly: `worldToGrid(gridToWorld(x,y)) ≈ (x,y)`

### AC-2: Gap-free rendering
- [ ] No sub-pixel tears or gaps between tiles at any camera scroll position
- [ ] A solid fill covers the full world rect so the canvas background is never visible through the grid

### AC-3: Grid lines
- [ ] Diamond outlines rendered on top of the fill at reduced opacity to show cell boundaries
- [ ] Grid covers at least 40×40 cells (configurable via `GRID_COLS` / `GRID_ROWS`)

### AC-4: Depth ordering
- [ ] Terrain renders at depth 0 (always behind trees and UI)
