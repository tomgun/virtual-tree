# F-0004: Tree Placement System — Acceptance Criteria

## Context
Players click the terrain to plant trees. Each tree must snap to the grid, be depth-sorted correctly, and grow over time.

## Acceptance Criteria

### AC-1: Click-to-plant
- [ ] Clicking anywhere on the terrain places a tree snapped to the nearest grid cell
- [ ] Clicks on UI elements (toolbar, minimap, help button, name dialog) do NOT plant a tree
- [ ] A player name must exist before a tree can be planted (name dialog shown if missing)

### AC-2: Multiple species
- [ ] At least 5 distinct species available (Oak, Pine, Palm, Cherry, Birch)
- [ ] Each species has a unique visual shape and colour palette
- [ ] Toolbar at the bottom allows switching species; keyboard shortcuts 1–5 also work
- [ ] Selected species is visually highlighted in the toolbar

### AC-3: Depth sorting
- [ ] Trees further from the viewer (higher grid Y) render behind closer trees
- [ ] Trees always render in front of the terrain layer

### AC-4: Growth
- [ ] Trees grow from sapling to mature over a configurable time period
- [ ] Test mode: full growth in 10 minutes; production mode: full growth in 10 days
- [ ] Visual size of the tree scales with its age/growth stage

### AC-5: Player tagging
- [ ] Each tree stores the planting player's name
- [ ] Species and player name persist across page refreshes (see F-0007)
