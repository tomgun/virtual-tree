# F-0009: Responsive UI Layout — Acceptance Criteria

## Context
The game UI must remain fully usable when the browser window is resized.

## Acceptance Criteria

### AC-1: Toolbar stays at bottom-centre
- [ ] Tree selector toolbar remains horizontally centred at the bottom edge after any resize
- [ ] Screen-space hit-rects for toolbar buttons update to match new positions

### AC-2: Top-right elements stay top-right
- [ ] Help (?) button stays anchored to the top-right corner
- [ ] Minimap stays below the help button, aligned to the right edge

### AC-3: No element clipping
- [ ] No UI element is pushed off-screen or overlaps another at common viewport sizes:
  - 1920×1080, 1440×900, 1280×720, 768×1024

### AC-4: Instructions panel
- [ ] Instructions are hidden by default behind a `?` button
- [ ] Clicking `?` toggles the panel; button changes to `✕` when open
