# F-0007: LocalStorage Persistence — Acceptance Criteria

## Context
Game state must survive page refreshes so players don't lose their trees.

## Acceptance Criteria

### AC-1: Save on change
- [ ] Game state (player name + all trees) saved to `localStorage` after every tree placement
- [ ] State also saved after every growth tick

### AC-2: Load on startup
- [ ] On page load, saved trees are restored and rendered at their persisted positions and ages
- [ ] Trees placed outside the current grid bounds (stale data) are silently skipped

### AC-3: Availability guard
- [ ] If `localStorage` is unavailable (private browsing, quota exceeded), the game continues without crashing

### AC-4: Schema
- [ ] Stored JSON contains: `playerName` (string), `trees` (array), `lastUpdated` (timestamp)
- [ ] Each tree entry contains: `id`, `x`, `y`, `age`, `species`, `playerName`, `plantedAt`
