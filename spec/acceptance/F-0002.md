# F-0002: Basic Phaser 3 Project Structure — Acceptance Criteria

## Context
A minimal Phaser 3 game must boot and render a canvas before any game-specific features are layered on.

## Acceptance Criteria

### AC-1: Game boots
- [ ] `index.html` loads; Phaser canvas is visible within 3 seconds on a modern browser
- [ ] Browser console shows no uncaught errors on initial load
- [ ] Background colour matches the grass palette (`#4a8038`)

### AC-2: Scene lifecycle
- [ ] `MainScene` registered and active; `create()` and `update()` lifecycle methods present
- [ ] Phaser input system enabled (keyboard + pointer) with `input.enabled = true`

### AC-3: Camera
- [ ] Camera bounds set to the full isometric world dimensions
- [ ] Camera starts centred on the middle of the grid
