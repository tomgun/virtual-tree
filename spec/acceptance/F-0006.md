# F-0006: Player Name Input and Tagging — Acceptance Criteria

## Context
Players identify themselves with a name so their trees can be attributed and shown on future leaderboards.

## Acceptance Criteria

### AC-1: Name input dialog
- [ ] Dialog auto-shown on first visit (no stored name)
- [ ] Space bar opens the dialog at any time when no other dialog is active
- [ ] Dialog contains a text input, a Save button, and a Skip button
- [ ] Pressing Enter in the input field saves the name; Escape dismisses with the previous name
- [ ] Clicking outside the dialog dismisses it (saves current or default name)

### AC-2: Name validation
- [ ] Empty name saved as "Player"
- [ ] Name trimmed of leading/trailing whitespace

### AC-3: Display
- [ ] Current player name shown in the top-left HUD below the CO₂ score
- [ ] Name updates immediately in the HUD after saving
