# F-0015: MainScene Refactor — Acceptance Criteria

## Context
`MainScene.ts` has grown to ~700 lines. The 500-line gate is bypassed on every commit.
Splitting it into focused modules improves testability, readability, and maintainability.

## Proposed modules
- `src/scenes/ui/HUD.ts` — score, player name, help button, instructions panel
- `src/scenes/ui/TreeSelector.ts` — toolbar, selector rects, species selection
- `src/scenes/ui/InfoPanel.ts` — CO₂ impact panel (buildInfoPanel, toggleInfoPanel)
- `src/scenes/input/InputHandler.ts` — all keyboard and pointer setup

## Acceptance Criteria

### AC-1: No regressions
- [ ] All existing features work identically after the refactor
- [ ] `npx tsc --noEmit` passes with 0 errors
- [ ] No new linter warnings introduced

### AC-2: File size
- [ ] `MainScene.ts` is ≤ 300 lines after extraction
- [ ] No extracted module exceeds 200 lines

### AC-3: Tests
- [ ] At least one unit test added per extracted module
